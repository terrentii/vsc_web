# vsc_web

Анонимный веб-мессенджер. Без слежки. Без истории по умолчанию.
Написан на Flask. Работает везде, где есть Python.

Живая версия: https://soufos.ru

---

    СТЕК        Flask · SQLAlchemy · Flask-WTF · Gunicorn
    ХРАНИЛИЩЕ   SQLite (комнаты, сообщения, пользователи, сессии)
    АВТОРИЗАЦИЯ Werkzeug хеширование паролей (scrypt)
    РЕАЛТАЙМ    AJAX-опрос каждые 3 секунды
    УВЕДОМЛЕНИЯ Service Worker (фоновые push)

---

## ПОЛЬЗОВАТЕЛИ

Два типа. Без смешения.

**Анон** — приходит без регистрации. Получает постоянный ID на основе
отпечатка IP + User-Agent: `Anon1`, `Anon2` и т.д. Номер никогда не
повторяется. Сменил IP или браузер — новый номер, навсегда.
Может читать и писать в открытых комнатах. Не может создавать комнаты.
Не может войти в закрытую комнату. Следов в списке участников не оставляет.

**Зарегистрированный** — выбирает логин один раз. Логин постоянный.
Может создавать комнаты, вступать в закрытые по приглашению,
управлять своими комнатами как Прародитель.
Ограничения логина: 3–32 символа, буквы (в т.ч. кириллица), цифры, нижнее
подчёркивание. Имена вида `Anon\d+` зарезервированы и отклоняются.

## КОМНАТЫ

**Открытая** — войдёт любой по 10-значному ID. Включая анонов.

**Закрытая** — только зарегистрированные, и только те, кого явно добавил
Прародитель. Аноны не входят. Исключений нет.

ID комнаты — всегда 10 цифр, генерируется случайно при создании,
уникальность гарантирована. Потерял ID — комната недоступна.

---

## ПРАРОДИТЕЛЬ

Зарегистрированный пользователь, создавший комнату.
Управляет: названием, типом (открытая/закрытая), списком участников
(добавить / выгнать). Не может быть выгнан. Не может покинуть.
Удаляет комнату или не передаёт никому.

## ЗАПУСК

    git clone https://github.com/terrentii/vsc_web
    cd vsc_web
    python -m venv .venv && source .venv/bin/activate
    pip install -r requirements.txt
    python app.py

Запускается на `http://localhost:5000`. Debug включён по умолчанию.

Продакшн:

    SECRET_KEY=ваш-ключ gunicorn server:application

Nginx-прокси обязан пробрасывать `X-Forwarded-Proto` — иначе CSRF
не работает поверх HTTPS.

## БЕЗОПАСНОСТЬ

- Пароли: scrypt через Werkzeug
- CSRF: Flask-WTF на всех изменяющих маршрутах
- Брутфорс логина: 5 попыток за 10 минут с одного IP
- Загрузка файлов: белый список MIME-типов, UUID-префикс имён, блокировка path traversal
- Доступ к медиа: проверка членства в комнате на каждый запрос файла
- Без сырого SQL — только ORM
- Секретный ключ: сохраняется в `.secret_key` при первом запуске, не пересоздаётся
