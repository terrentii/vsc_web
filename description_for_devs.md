# Структура проекта: Анонимный мессенджер

---

## 1. Пользователи — два типа

### Анон
- При входе на сайт автоматически получает временный ID: `Anon1`, `Anon2` и т.д. (порядковый номер генерируется случайно или инкрементально)
- ID не сохраняется между сессиями — при перезаходе будет другой ID
- Связать реального человека с AnonID невозможно
- **Возможности:** читать открытые комнаты, писать в открытые комнаты
- **Ограничения:** не может создавать комнаты, не может входить в закрытые комнаты, не записывается в `users.csv` комнаты

### Зарегистрированный пользователь
- Выбирает уникальный логин при регистрации — он постоянный и привязан к аккаунту
- Логин теоретически можно связать с личностью (пользователь должен это понимать)
- **Дополнительные возможности:** создавать комнаты, входить в закрытые комнаты (если записан в `users.csv` этой комнаты), получать статус Прародителя

---

## 2. Комнаты — два типа

### Открытая комната
- Доступна всем — и анонам, и зарегистрированным
- Любой может читать и писать сообщения
- Аноны в `users.csv` **не записываются**

### Закрытая комната
- Читать и писать могут только зарегистрированные пользователи, записанные в `users.csv` этой комнаты
- Аноны не имеют доступа вообще
- Тип комнаты (открытая/закрытая) хранится в `config.csv`

---

## 3. Структура файлов комнаты (3 CSV)

Каждая комната — это папка или набор файлов с уникальным ID комнаты.

### `config.csv`
| Поле | Описание |
|---|---|
| `room_id` | Уникальный идентификатор комнаты |
| `room_name` | Название, заданное Прародителем |
| `is_open` | `true` / `false` — тип комнаты |
| `created_at` | Дата создания |
| `creator_login` | Логин Прародителя |

### `messages.csv`
| Поле | Описание |
|---|---|
| `author` | Логин или AnonID |
| `timestamp` | Дата и время сообщения |
| `text` | Текст сообщения |

Сообщения отображаются **хронологически** — от первого до последнего, как надписи на стене.

### `users.csv`
| Поле | Описание |
|---|---|
| `login` | Логин пользователя |
| `joined_at` | Дата вступления |
| `role` | `founder` (Прародитель) или `member` |

- Первая строка — всегда Прародитель
- Аноны сюда **никогда не попадают**
- Именно этот файл является «пропуском» в закрытую комнату

---

## 4. Прародитель — особая роль

- Первый зарегистрированный пользователь, создавший комнату
- Записан первым в `users.csv` с ролью `founder`
- **Дополнительные полномочия** (логично предположить из структуры):
  - Менять название комнаты
  - Менять тип комнаты (открытая ↔ закрытая)
  - Добавлять/удалять пользователей из `users.csv` (управление доступом)

---

5. Логика входа на сайт (обновлено)
Главная страница предлагает два действия:
Подключиться к комнате

Пользователь вводит 10-значный цифровой ID комнаты вручную
Система проверяет существование комнаты
Далее — проверка доступа (открытая/закрытая, см. ниже)
Если комнаты с таким ID не существует → ошибка, предложить попробовать снова

Создать комнату

Только для зарегистрированных пользователей
Система генерирует случайный свободный 10-значный ID
Пользователь автоматически становится Прародителем и попадает в комнату
Комната создаётся с дефолтными настройками (открытая)

Главная страница
      │
      ├─── [Подключиться] → ввод 10-значного ID
      │                          │
      │                    Комната есть?
      │                    /          \
      │                  ДА           НЕТ → ошибка
      │                   │
      │             Комната открытая?
      │               /          \
      │             ДА            НЕТ
      │              │             │
      │           Войти      Залогинен + в users.csv?
      │                         /       \
      │                       ДА        НЕТ → отказ
      │                        │
      │                      Войти
      │
      └─── [Создать] → только залогиненный
                            │
                     Генерация свободного ID
                            │
                     Создание трёх CSV
                            │
                  Пользователь = Прародитель → войти

Название комнаты — дополнительная задача (низкий приоритет)

Прародитель может задать название после создания комнаты
Если название не задано — отображается просто ID комнаты
Хранится в config.csv, поле room_name (может быть пустым)
Реализуется в последнюю очередь, на основную логику не влияет


Всё остальное из предыдущей структуры (типы пользователей, CSV-файлы, права доступа, маршруты) остаётся без изменений.

## 6. Взаимосвязи — сводная таблица прав

| Действие | Анон | Зарегистрированный | Прародитель |
|---       |---   |---                 |          ---|
| Войти в открытую комнату | ✅ | ✅ | ✅ |
| Войти в закрытую комнату | ❌ | ✅ (если в users.csv) | ✅ |
| Писать в открытую комнату | ✅ | ✅ | ✅ |
| Писать в закрытую комнату | ❌ | ✅ (если в users.csv) | ✅ |
| Создать комнату | ❌ | ✅ | — |
| Управлять комнатой | ❌ | ❌ | ✅ |
| Записываться в users.csv | ❌ | ✅ | ✅ |
| Менять свой ID при перезаходе | ✅ (автоматически) | ❌ (логин постоянный) | ❌ |

## 7. Технический стек и архитектура Flask

- **Flask** — основной фреймворк
- **Flask-Session** или cookies — хранение типа пользователя (анон/залогинен) на стороне сервера
- **CSV** — хранилище данных (по одной папке на комнату)
- **Flask-SocketIO** (опционально) — если нужна живая подгрузка сообщений без перезагрузки страницы, иначе достаточно обычных GET/POST запросов

### Ключевые маршруты
```
GET  /                  → главная, форма ввода ID комнаты
POST /room/join         → логика входа/создания комнаты
GET  /room/<room_id>    → страница комнаты
POST /room/<room_id>    → отправка сообщения
GET  /register          → регистрация
POST /register          → создание аккаунта
GET  /login             → вход
POST /login             → аутентификация
POST /room/create       → создание комнаты (только залогиненный)
POST /room/<id>/manage  → управление комнатой (только Прародитель)
```
